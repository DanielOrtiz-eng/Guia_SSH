Configuración SSH para GitHub

Guía paso a paso para autenticación sin contraseñas

1
Crear carpeta SSH
mkdir ~/.ssh cd ~/.ssh
Ubicación: C:\Users\TuUsuario\.ssh
2
Generar llave SSH
ssh-keygen -t ed25519 -C "tuemail@ejemplo.com"
IMPORTANTE: Usa el mismo email que en GitHub
PRESIONAR: ENTER 3 veces
3
Crear archivo config
notepad ~/.ssh/config
Host github.com HostName github.com User git IdentityFile ~/.ssh/id_ed25519
Esto le dice a SSH cómo conectarse con GitHub
4
Activar agente SSH
eval "$(ssh-agent -s)" ssh-add ~/.ssh/id_ed25519
El agente SSH guarda tu llave en memoria
5
Copiar llave pública
cat ~/.ssh/id_ed25519.pub
PRIVADA
id_ed25519
NUNCA compartir
PÚBLICA
id_ed25519.pub
SÍ compartir con GitHub
En GitHub: Settings → SSH and GPG keys → New SSH Key → Pegar y guardar
6
Probar conexión
ssh -T git@github.com
Va a preguntar: Escribir: yes
Si funciona verás: "Hi [tu-usuario]! You've successfully authenticated..."
7
Ir al proyecto
cd "Hello Git" git status
git status muestra cambios pendientes por guardar
8
Crear repositorio en GitHub
1. New Repository
2. Nombre: "Hello Git"
3. NO crear README
4. Copiar URL SSH
git@github.com:usuario/hello-git.git
9
Conectar y subir (push)
git remote add origin git@github.com:usuario/hello-git.git git remote -v git push -u origin main
remote add: Conecta repositorios
remote -v: Verifica conexión
push: Sube código a GitHub
10
Bajar cambios (Pull)
1. Haz un cambio en GitHub.com (edita README.md)
git pull
PUSH
Local → GitHub
git push
PULL
GitHub → Local
git pull
